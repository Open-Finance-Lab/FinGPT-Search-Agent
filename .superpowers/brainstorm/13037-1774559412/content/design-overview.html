System Architecture

End-to-End Data Flow
USER SIDE
chatgpt.com / claude.ai / gemini.google.com
User asks financial question → model responds
↓ User clicks "Validate"
Browser Extension
Mode Toggle: [Chat] [Validate]
• Validate mode: DOM parser extracts model output → sends to /validate/
• Chat mode: existing Agentic FinSearch chatbot (unchanged)
• Validation results feed into chat context for follow-ups
BACKEND
NEW: /validate/ endpoint
1. Receive raw model output text
2. Claim Decomposer → atomic claims
3. Claim Router → match claims to data sources
4. Claim Verifier → check against ground truth
5. Return structured validation report

What's New vs. What's Reused

✦ NEW COMPONENTS
  • DOM Parser — site-specific extractors for ChatGPT, Claude, Gemini
  • Claim Decomposer — LLM prompt to extract atomic claims
  • Claim Router — maps claim types to data sources
  • Claim Verifier — compares claims to ground truth
  • Validation UI — results panel with claim-level indicators
  • /validate/ endpoint — orchestrates the pipeline
  • Mode toggle — Chat ↔ Validate in extension
♻ REUSED AS-IS
  • MCP infrastructure — Yahoo Finance, TradingView, SEC EDGAR
  • Chat API — /get_chat_response/, /get_adv_response/
  • Session management — UnifiedContextManager
  • Planner/Skills — route follow-up questions
  • numerical_validator.py — upgraded into Claim Verifier
  • quality_logger.py — extended for validation metrics
  • Frontend chat UI — existing chat components
  • Django backend — settings, auth, middleware
